# 30-Tage-Plan

## Regeln der Challenge
- Kein GPT für Lösungen
- Nur Dokumentationen lesen, googlen, Debuggen
- Jeden Tag mindestens 30-60 Minuten
- Mini-Tagebuch führen (max. 5 Zeilen)

## Woche 1 - Basics
**Tag 1:** Variablen & Datentypen -> Name, Alter, Ziel ausgeben
**Tag 2:** Benutzerinput → Lieblingszahl + kleine Berechnung  
**Tag 3:** Strings → slicing, upper/lower, f-Strings, Vokale zählen  
**Tag 4:** Listen → To-Do-Liste erstellen & manipulieren  
**Tag 5:** Tupel & Sets → Duplikate entfernen  
**Tag 6:** Dictionaries → Videospiel-Charakter erstellen  
**Tag 7:** Mini-Datenbank → Liste von Dictionaries, CRUD  

## Woche 2 – Kontrolle & Modularität
**Tag 8:** if/else → Passwort-Checker  
**Tag 9:** for-Loops → Multiplikationstabelle  
**Tag 10:** while-Loops → Zahl-Raten-Spiel  
**Tag 11:** Funktionen Basics → Matheoperationen  
**Tag 12:** Funktionen II → Textanalyse-Funktion  
**Tag 13:** Fehler & Debugging → Input-Fehler abfangen  
**Tag 14:** Wochenprojekt → Taschenrechner mit Funktionen  

## Woche 3 – Aufbauwissen
**Tag 15:** Module & Imports → eigenes helpers.py  
**Tag 16:** Dateien → Tagebuch speichern & laden  
**Tag 17:** JSON → Datenstruktur speichern & laden  
**Tag 18:** Klassen Basics → Animal-Klasse  
**Tag 19:** Klassen II → BankAccount-Klasse  
**Tag 20:** Vererbung → Vehicle → Car, Bike  
**Tag 21:** Wochenprojekt → Text-RPG oder Adventure  

## Woche 4 – Projekte & Praxis
**Tag 22:** Algorithmen → Bubble Sort & Suche  
**Tag 23:** Time & Date → Alter berechnen  
**Tag 24:** Unit Tests → eigene Funktionen testen  
**Tag 25:** CLI Tools → Textanalyse-Tool  
**Tag 26:** Refactoring → Code sauber machen  
**Tag 27:** Git Basics → Repo erstellen & pushen  
**Tag 28:** Mini-Framework → tkinter / pygame Einstieg  
**Tag 29:** Projektwahl → Mini-Spiel, Datenbank, Notizen-App  
**Tag 30:** Abschlusstest → 80–150 Zeilen Projekt, alles kombiniert
